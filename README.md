# Claude-Certified-Engineer
A 4-week lesson plan based on the Claude Certified Architect study guide from https://github.com/paullarionov/claude-certified-architect

To get started, take a look at the [overview](overview.md) document.

---
## Study Tips
- Clone this repository and create a local branch called `{your name}_local`
- When you sit down to study each day, make sure to run a `git pull` to see if any changes have been made to the course content since you last checked. If there are changes on the `main` branch, be sure to merge them onto your local branch.
- As you work through the content on your own branch, write down your answers to each of the practice questions. This will help you remember the content much better than just thinking about it. 
- Create a new project in Cowork and give it access to the repo folder. Copy CLAUDE_TEMPLATE into your own CLAUDE.md and fill in your name/experience, so Claude can act as a tailored personal tutor.
- Whenever you find something unclear in the content, ask your personal tutor! When you understand the the concept, switch to a new git branch off of `main` and tell Claude "I understand". It will propose changes to the weekly document, and commit them once you approve. Feel free to do research online and/or make manual edits too!
	- **IMPORTANT:** DO NOT COMMIT CHANGES TO THE COURSE CONTENT ALONG WITH YOUR ANSWERS TO THE EXERCISES.
- After committing proposed changes to the course content, you can switch back to your local branch and continue working. You may merge your changes onto your local branch if you would like.
- When you finish the content for a day or a week and would like to publish your coursework suggestions, push your suggestions branch to GitHub and make a pull request. Once someone reviews your changes and accepts the pull request, the whole team can benefit from your suggestions and learning!
	- **IMPORTANT:** PULL REQUESTS CONTAINING ANSWERS TO THE PRACTICE QUESTIONS WILL NOT BE ACCEPTED. KEEP YOUR ANSWERS ON A SEPERATE BRANCH FROM THE WORKING COURSE CONTENT.
- Check GitHub regularly for pull requests from others. If you have already covered the content they have suggestions about, please review the pull requests so those following behind you can benefit from the course updates.
